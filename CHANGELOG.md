# Changelog

本專案的所有重要變更都記錄於此。格式依循 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.1.0/),版本遵循 [Semantic Versioning](https://semver.org/lang/zh-TW/)。

## [0.18.0] - 2026-07-24

對強制簽章模式的一輪對抗性審查與收緊:修掉一個讓惡意伺服器得以偷偷降級成員的反回滾缺陷。

### 安全

- **強制簽章反回滾改為 fail-closed**:先前成員每次重新取得團隊金鑰時,會用伺服器當下回傳的政策無條件覆蓋本機的強制態——惡意伺服器只要「抑制」(不回傳)政策,就能把已啟用強制簽章的成員偷偷降級回「容忍未簽章寫入」,重新打開注入窗口。現在成員明確區分三態:當代擁有者簽章政策的**開啟**、**關閉**、與**政策缺席**;唯有明確的當代政策才改變設定,缺席一律**保留本機既有狀態**,惡意伺服器抑制政策無法再造成降級。輪換時一律以新紀元重簽當前狀態(開啟或關閉),讓「關閉」也能正確跨紀元傳播,不會有成員永久卡在過期的強制態。
- 殘餘(信任源自首次接觸的本質限制,已載於 `SECURITY.md`):從未收過政策的成員——例如惡意伺服器自首次接觸就抑制——退回過渡容忍,與所有擁有者簽章狀態相同;以及持續在線的成員於下次重連或輪換才套用擁有者的切換(誠實伺服器的伺服器端拒收提供其間的即時保護)。

### 品質

- 政策 tri-state 端到端測試:政策未設/被抑制(直接刪除)回報「缺席」以保留 pin、開啟回報開啟、明確關閉回報關閉。全庫測試全綠。

## [0.17.0] - 2026-07-24

作者歸屬:留言作者從自我宣稱升級為密碼學可驗——標記「這則留言出自擁有者背書的合法成員」及其角色。

### 新增

- **可驗證的留言作者**:留言與回覆的作者改綁密碼學成員身分(`memberId`),而非先前的裝置代號。收件端對照擁有者背書的成員憑證目錄,在作者是當紀元合法成員時顯示「✓ 角色」徽章;裝置代號的舊留言、未知或已移除的作者則不標記。團隊成員變動(核准、移除、輪換)後徽章即時刷新。

### 安全

- 留言本身就是 doc 寫入,自 0.16 起已逐筆簽驗:唯讀成員或非成員的留言寫入在同步層即被拒,徽章只是把這層既有的真實性浮到介面。徽章表示「作者是擁有者背書的成員」;顯示名仍為成員自選標籤,合法編輯者在內容中冒填他人身分屬稽核範疇(見 `SECURITY.md` §4.1),非本功能所擋。
- 明確不做逐字 blame(CRDT 合併壓縮後 op 對作者的映射會碎裂);作者歸屬止於寫入層級。

### 品質

- 端到端斷言:成員同步後其已驗證目錄含擁有者(owner)與協作者(editor)及正確角色,涵蓋徽章的資料來源路徑。全庫測試全綠。

## [0.16.0] - 2026-07-23

寫入真實性收官:團隊 vault 的每筆編輯都由作者簽章、收件端驗證,並可由擁有者切換強制模式徹底拒收未簽章寫入——「誰寫了什麼」從社交信任升級為密碼學可驗。

### 新增

- **每筆寫入作者簽驗**:團隊 vault 的每筆增量與快照都帶作者的 Ed25519 簽章(簽密文雜湊,收件端先驗作者再解密)。收件端查成員憑證目錄取得作者可信公鑰、確認其為當紀元的擁有者/編輯者、驗簽通過才套用;驗不過的寫入(惡意中繼注入、被移除者殘留、唯讀成員寫入)沿用既有 poison-skip 丟棄並跨越,不卡序列。
- **強制簽章模式**:擁有者可簽發一份綁定金鑰紀元的 vault 政策開啟「拒收未簽章寫入」,關閉升級過渡期對舊用戶端的容忍窗口。政策隨金鑰信封原子下發、成員對擁有者公鑰驗證後套用並本機留存(抵抗惡意伺服器事後抑制);伺服器亦一併軟拒未簽章寫入作縱深防禦。團隊面板新增擁有者專屬開關。

### 安全

- 唯讀成員的「不得寫入」從可信伺服器邊界升級為**收件端可驗**:即使惡意伺服器願意中繼唯讀成員或被移除者的寫入,每個收件端都會因作者角色/紀元不符而拒收。
- 作者簽章綁定 `{寫入類型, docId, 金鑰紀元, 密文雜湊}`,不綁序號/裝置(轉發不帶、CRDT 套用冪等使重放無害);跨 doc 挪用、跨紀元重放、竄改內容一律驗不過。
- vault 政策綁金鑰紀元,輪換時以新紀元重簽,避免輪換後強制態靜默失效;偽造、跨 vault 挪用、竄改旗標一律驗不過。過渡窗口與殘餘威脅(伺服器僅能抑制可用性、合法編輯者的內容屬稽核範疇)已明列於 `SECURITY.md`。

### 品質

- 逐 update 簽章與 vault 政策的單元測試(正簽正驗、偽簽、跨 vault、跨紀元、竄改、截斷全拒),以及兩支端到端金牌:合法協作在簽驗/強制下照常收斂;偽造作者與未簽章注入(即使以團隊金鑰加密解得開)一律被拒不套用。全庫測試全綠。
- 桌面整合測試改為序列執行,消除多支端到端測試並行時互相爭搶 CPU 造成的非邏輯性 flaky。

## [0.15.0] - 2026-07-23

成員憑證:團隊成員的簽章公鑰由擁有者背書、可驗,為「寫入作者驗證」鋪好地基。

### 新增

- **成員憑證與可信目錄**:擁有者在核准成員、改角色或輪換金鑰時,簽發一份綁定「這位成員的簽章公鑰 ↔ 成員 id」的憑證。任一成員都能拉取全團隊的成員憑證目錄,逐筆對擁有者公鑰驗證——從此能密碼學地確認「這個成員 id 對應的簽章公鑰確實是擁有者認可的」。這是後續「每筆編輯的作者可驗」的前置基建;目錄本身也讓成員能驗證彼此身分。

### 安全

- 成員 id 由簽章公鑰導出(`sha256(pubSign)`),憑證不單獨簽 id、由驗證者自算;團隊 id 由驗證者自帶。惡意中繼捏造假成員、竄改某成員的公鑰、或跨團隊挪用憑證,一律驗不過而被目錄濾除。憑證綁定金鑰紀元,輪換即作廢整代。
- 目錄拉取開放給任一認證成員(簽章公鑰本是公開資訊,真實性由每筆憑證的擁有者簽章保證);簽發仍限擁有者。

### 品質

- 成員憑證單元測試(正簽正驗且成員 id 由公鑰導出、非擁有者簽、跨團隊、公鑰/角色竄改、截斷全拒)與端到端測試(一般成員拉目錄取得全員可信公鑰、惡意摻假被濾、移除成員後從目錄消失)。全庫測試維持全綠。

## [0.14.0] - 2026-07-23

真機走查後的一輪修復:受限空間更誠實、檢視者真唯讀、團隊流程更順,並修掉數個同步收斂與安全問題。

### 新增

- **空間內直接新建筆記**:側欄每個空間標題列新增「＋」,建立的筆記自動歸入該空間,省去「先建再移」。
- **被移出團隊的明確提示**:被擁有者移除後,側欄顯示紅色橫幅、同步指示轉紅並停止重連,不再靜默無感。

### 變更

- **待核准免重開**:以邀請碼加入後,擁有者一核准即自動開始同步,不必手動重開 vault。
- **加入團隊帶顯示名**:建立或加入團隊時預設帶入系統使用者名,協作游標不再一律顯示「訪客」。
- **名單外成員看不到受限空間**:受限空間及其筆記在無權成員的側欄完全隱藏;先前已同步的本地檔案在失去授權時移入回收桶(受限前已在對方裝置上的明文屬已知限制,無法回收)。

### 修復

- **檢視者真唯讀**:檢視者角色的編輯器與留言收斂為唯讀,不再出現「能編輯但同步不上去」的假象;伺服器對唯讀成員的寫入改為軟拒,避免離線編輯重連時陷入斷線迴圈。
- **空間存取變更即時反映**:授權或撤銷後,名單內成員免重開自動補齊內容、名單外成員的側欄即時更新。
- 團隊面板與空間相關對話框的多處排版與按鈕樣式修正。

### 安全

- **受限空間金鑰不外洩**:受限與否改以金鑰信封層為權威判定(與金鑰原子取得),消除「名單已同步、金鑰未到」的競態窗口下以團隊金鑰誤加密受限內容的風險;加密端加上「無該空間金鑰即拒絕加密」的硬性防線。
- **輪換重加密不被靜默略過**:金鑰輪換撞上既有快照點時,以新金鑰的密文為準覆蓋,避免舊金鑰快照殘留。
- **失去授權的本地移除不誤傳共享刪除**:無權而被移入回收桶的筆記只卸下同步,絕不從共享中繼資料刪除路徑,免得連帶刪掉名單內成員(含擁有者)的筆記。

### 品質

- **同步收斂修復**:快照更新現在會廣播給同 vault 其他連線,落後成員不再因增量被截斷而卡在舊進度;跳號補洞加上節流,避免自我再生的緊迴圈;解不開的增量以本地快照跨越,不再永久卡住序列。全庫測試維持全綠。

## [0.13.0] - 2026-07-23

空間存取:某個空間只給部分成員——受限空間有自己的金鑰,名單外的人拿到密文也解不開。

### 新增

- **空間成員子集**:團隊 vault 的擁有者可把任一自訂空間設為「受限」,勾選哪些成員能存取。受限空間使用**獨立的隨機金鑰**(不由團隊金鑰衍生),只包給名單內成員與擁有者——名單外成員即使持有團隊金鑰,對該空間的密文也無能為力。這是密碼學邊界,不是介面隱藏。
- **名單外成員的乾淨體驗**:受限空間的筆記在名單外成員的裝置上完全不落地——新筆記不出現、不下載、無解密錯誤;受限前已同步的檔案停在最後可讀狀態。之後獲授權,內容自動補齊。
- **團隊面板「空間存取」**:每個空間顯示「全團隊」或「限 N 人+擁有者」,勾選成員後一鍵「儲存並輪換金鑰」;隨時可恢復開放全團隊。

### 變更

- **存取變更即輪換**:授權、撤銷、恢復開放都伴隨一次金鑰輪換——每次輪換為每個受限空間換發新金鑰、只重包給當下名單,被撤銷者的舊空間金鑰立即對新內容失效。沿用 0.11.0 輪換管線的全部保證(寫入柵欄、崩潰續跑、成員自癒)。
- **留言跟著空間走**:筆記的留言現在與筆記本體用同一把空間金鑰加密——受限筆記的留言同樣只有名單內成員讀得到(先前留言一律以團隊金鑰加密)。

### 安全

- **前向限制(明確界定)**:把「已有內容」的空間轉為受限,只保護此後的內容——受限前的內容,名單外成員的裝置上早已持有明文副本與舊密文。要放機密內容,先建受限空間、再把內容放進去。

### 品質

- 金牌測試(三成員經真伺服器):名單內成員以空間金鑰收斂編輯;名單外成員新筆記不落地、既有檔案停更、以團隊金鑰解伺服器密文被拒;撤銷後舊空間金鑰對新內容失效、擁有者恆可解。另有空間金鑰衍生、名單模型、伴生 doc 歸屬的單元測試。全庫 436 測全綠。

## [0.12.0] - 2026-07-22

角色憑證:成員的角色由擁有者簽章、客戶端可驗,伺服器無法捏造。

### 新增

- **客戶端可驗的角色憑證**:擁有者核准成員、改角色或輪換金鑰時,都會對「這個 vault、這位成員、這個角色、這個金鑰紀元」簽出一份 Ed25519 憑證,經伺服器分發;成員取金鑰時以邀請時取得的擁有者公鑰(信任錨)驗證。**角色指派從此防竄改**——被攻陷的伺服器無法把檢視者謊稱成編輯者,補上 0.10.0 明定的信任邊界缺口。

### 安全

- **偽造即拒**:憑證簽章驗不過(偽簽、跨 vault 或跨成員挪用、內容竄改)時,成員端直接中止金鑰載入,不會拿著假角色運作。
- **舊憑證重放防護**:憑證綁定金鑰紀元,輪換金鑰即作廢整代舊憑證——伺服器重放輪換前的憑證不會被採信。同一紀元內改角色後的舊憑證重放仍屬已知限制(兩者皆為真簽),擁有者輪換金鑰即可沖洗;每筆編輯的作者驗證留待後續版本。
- **平滑升級**:升級前已核准的成員尚無憑證,維持原行為(顯示本地既知角色);擁有者下次改角色或輪換金鑰時自動補發。

### 品質

- 憑證單元測試(正簽正驗、偽簽、挪用、內容竄改、截斷)與端到端測試(核准/改角色簽發流、資料庫植入偽簽憑證即拒、輪換重簽與舊紀元重放降級、非擁有者簽發被拒、移除成員清除憑證)。全庫 429 測全綠。

## [0.11.0] - 2026-07-22

金鑰輪換:移除成員後整個 vault 換新金鑰並重新加密,離線舊金鑰從此解不開新內容。

### 新增

- **移除成員即輪換金鑰**:擁有者移除成員後,自動產生新的團隊金鑰、重新包給所有留任成員,並把 vault 內容全部以新金鑰重新加密。被移除者即使留有離線舊金鑰與舊快取,也解不開此後的任何內容——補上 0.10.0 已知限制中的密碼層前向保密。
- **手動輪換**:團隊面板新增「輪換金鑰」按鈕,自動輪換失敗(例如網路中斷)時可重試,也可作例行輪換。
- **待核准標記**:成員清單標示尚未核准的成員;輪換只把新金鑰重新包給已核准成員,不會繞過指紋核對。

### 變更

- **輪換後分享連結作廢**:分享連結內含由團隊金鑰衍生的筆記金鑰,輪換後一律失效並即時切斷已連線的分享檢視;需要時以新金鑰重新建立分享。個人 vault 的分享不受影響。

### 安全

- **紀元寫入柵欄**:每個團隊 vault 帶單調遞增的金鑰紀元(epoch),輪換 commit 後伺服器立即拒絕舊紀元的寫入——確保輪換窗口內不會有舊金鑰密文混入共享日誌造成永久污染。
- **崩潰安全**:輪換全程冪等——commit 前任何失敗都不留半套狀態(紀元不動、舊金鑰續用);commit 後若重加密中斷,擁有者重啟會自動續跑補完(每篇重加密都等伺服器確認落盤才算數)。
- **成員自癒**:線上成員收到輪換廣播即暫停推送、取回新金鑰後無縫收斂;離線錯過輪換的成員重連時發現紀元落後,同樣自動取回新金鑰並重新拉齊,期間絕不套用解不開的資料。

### 品質

- 輪換全鏈測試:伺服器層(紀元 CAS 只接受恰為當前+1、柵欄拒舊紀元寫入、輪換廣播、分享作廢與踢線)與桌面金牌(三成員經真伺服器——留任者收斂新金鑰並雙向編輯、被移除者舊金鑰解不開重加密內容、重加密冪等、離線者重連自癒、未拉齊即中止且紀元不動)。全庫 419 測全綠。

## [0.10.0] - 2026-07-21

團隊角色與授權:owner / editor / viewer,伺服器強制讀寫權,踢人立即生效。

### 新增

- **成員角色(owner / editor / viewer)**:團隊 vault 現在區分角色。**擁有者**全權並可管理成員;**編輯者**可讀寫;**檢視者**唯讀。擁有者產生邀請碼時就決定被邀者的角色,加入即生效。桌面團隊面板可顯示每位成員的角色、隨時把成員在編輯者/檢視者間切換,或移除。
- **伺服器逐訊息授權**:團隊 vault 上,只有擁有者/編輯者能寫入(含改動筆記路徑、空間等共享結構);檢視者的任何寫入一律被伺服器拒絕,但仍可讀取與即時看到協作。個人 vault 完全不受影響。
- **移除/降級立即踢線**:擁有者移除成員或把編輯者降為檢視者時,對方的**活躍連線當場切斷**——不再等到下次斷線。被移除者無法重連(邀請碼一次性、已消耗)。

### 變更

- 升級既有 0.9.0 團隊 vault 時,伺服器會自動為成員名冊補上角色欄位(既有成員預設為檢視者,擁有者維持擁有者身分);**擁有者可在團隊面板重新指派各成員的角色**。

### 安全

- **授權的信任邊界(明確界定)**:在全盲中繼的端對端加密模型下,角色的**讀寫強制是「可信伺服器」層級的控制**(防濫用、防誤操作),而非密碼學保證——檢視者本身持有團隊金鑰,一個被攻陷的伺服器理論上仍可接受其寫入。**機密性與金鑰分發**維持不信任伺服器的端對端加密(空間金鑰信封的擁有者簽章才是客戶端可驗的密碼學邊界)。把角色指派也變成客戶端可驗、以及每筆編輯的作者驗證,留待後續版本。

### 品質

- 角色全鏈測試:store 角色 CRUD 與邀請碼帶角色、協議往返;伺服器端對端(編輯者可寫 / 檢視者寫入含 vault-meta 皆拒 / 任何成員可讀、移除與降級即時踢線並擋重連、非擁有者管理指令被拒、升級自癒讓遷移後的擁有者仍能寫入)。全庫 404 測全綠。

### 已知限制(留待後續)

- **踢人尚未輪換金鑰**:移除成員只切連線與刪其金鑰信封,不重換團隊金鑰;被移除者若留有離線舊快取,仍能解先前已同步的舊內容(對「新內容」則因無法重連而取不到)。密碼層的前向保密(換金鑰 + 重加密)為下一階段。

## [0.9.0] - 2026-07-21

團隊協作正式落地:多人各以自己的身分,共享同一個加密 vault。

### 新增

- **團隊 vault(邀請 + 空間金鑰盲中繼)**:一個團隊 vault 有一把隨機的**團隊金鑰**,用每位成員的公鑰包成信封,經**全盲伺服器**轉交;成員以自己的身分金鑰解出後即可協作。伺服器看不到金鑰明文,也讀不到內容。加入流程:擁有者產生**一次性邀請碼**交給對方(out-of-band),對方憑碼加入後,擁有者**核對其公鑰指紋**再核准(把團隊金鑰包給他)。核准前成員處於「等待授權」狀態,不會誤讀任何東西。擁有者可**移除成員**。桌面新增團隊面板:建立 / 加入團隊 vault、產生邀請碼、成員清單與指紋核對、核准 / 移除。
- **身分種子上 OS keychain**:成員身分的種子改由系統金鑰鏈(macOS Keychain / Windows DPAPI)加密後才落盤;keychain 不可用時優雅退回、並相容既有明文檔。因為這把種子的洩漏面在團隊時代跨越多個 vault,值得比 vault 密語更強的保護。

### 變更

- **團隊 vault 強制身分認證**:設有擁有者的 vault 一律走 challenge-response 身分認證,拒絕只帶 token 的舊式連線,並擋下非成員的寫入——順帶堵掉「任何持 token 者上傳一份垃圾快照即可截斷他人真實編輯」的可用性攻擊。個人 vault 的密語同步完全照舊,零改變。

### 安全

- **信封由擁有者簽章 + 情境綁定**:空間金鑰信封帶擁有者的 Ed25519 簽章,成員以 out-of-band 已知的擁有者指紋驗簽——擋下「惡意伺服器自造信封 + 自造中繼資料,餵新成員一個偽造的 vault」。信封的加密情境綁定 vault / 金鑰 / 版本 / 收件人,防跨情境挪用重放;解封拒絕退化的金鑰交換結果。

### 品質

- 團隊全鏈測試:金鑰信封 wrap/unwrap(含偽造、竄改、跨情境、退化點防線）、協議往返、伺服器授權(邀請碼單次 / 綁 vault / 過期、擁有者專屬操作、信封只回本人、team vault 拒舊式連線)、bootstrap 死結防護與 pending→ready、擁有者管理連線、邀請 bundle 編解碼、身分種子 keychain 往返;**金牌**:兩個不同身分的成員經真伺服器雙向收斂、以團隊金鑰加解密互通。全庫 393 測全綠。

### 已知限制(留待後續)

- **踢人後金鑰未輪換**:移除成員只刪其名冊列與信封,不重換團隊金鑰;被移除者若留有舊金鑰快取,仍能離線解舊內容。真正的踢人(輪換 + 重包給留任者）與 owner/editor/viewer 角色強制留待下一階段。
- **per-space 成員子集**:本版團隊是「整個 vault 共享」,尚無「某空間只給部分人」。資料模型已為此預留。

## [0.8.0] - 2026-07-21

本地優先補完 + 團隊協作的身分地基:空間與留言不再需要同步,並為團隊功能鋪好成員身分。

### 新增

- **成員身分(團隊協作的地基)**:每台裝置持有一把長期的**非對稱成員身分**——單一隨機種子衍生出簽章(Ed25519)與金鑰包裝(X25519)兩把子金鑰。連線時對伺服器做 **challenge-response**(簽名綁定伺服器亂數 + vault + 成員,防重放與跨 vault 挪用),證明「你是誰」而不只是「你知道 token」。伺服器把成員公鑰記進 per-vault 名冊(公鑰**首見即釘選**,擋冒名)。身分獨立於任何 vault 密語,存本機獨立檔、可匯出匯入到其他裝置。**這是團隊空間的樞紐**:未來邀請成員 = 用對方的公鑰把空間金鑰包起來經全盲伺服器轉交,難題從「不可能」降到「可做」。個人使用者無感——現有 vault 的密語同步完全照舊。

### 變更

- **空間與留言不再需要啟用同步**:先前「空間分組」與「留言討論」都掛在同步層管理的中繼資料上,未開同步時整組退化(側欄變平面清單、留言面板停用)。現在把中繼資料的所有權從同步層抽離,交由 vault 本身持有——**純本地、沒開同步的 vault 一樣能建空間、移動/複製筆記、留言討論**,同步啟用時才額外把它們推上雲。這修正了本地優先產品不該有的破口。

### 品質

- 身分全鏈測試:金鑰生成/簽章/序列化 6 例、協議往返、伺服器 challenge-response 端對端(含搶註、重放、跨裝置匯入)、桌面身分檔持久化;純本地空間 5 例 + 純本地留言 2 例守住「無同步也可用」不變量。
- 安全審查(密碼學 + 認證):修掉一個搶註漏洞(伺服器綁定成員 id 為其公鑰雜湊,擋任意 id 配自帶金鑰霸佔他人身分),並硬化身分檔落盤(原子寫、壞檔不靜默換身分)。桌面測試增至 97,全庫 347 測全綠。

### 已知限制

- 團隊空間本身(邀請、角色、用公鑰包裝空間金鑰)尚未開放——本版只鋪身分地基,伺服器記錄成員但**尚不以成員身分做授權**,存取仍靠既有 token + 密語。成員名冊在本版是諮詢性紀錄。

- **自架同步伺服器**:Node + ws + better-sqlite3 的 blind relay,單 Docker 一行起服;雙路徑同步(快照 + 增量)、離線佇列、路徑 LWW。
- **i18n**:zh-TW / en,CI 缺譯檢查。
