# Changelog

本專案的所有重要變更都記錄於此。格式依循 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.1.0/),版本遵循 [Semantic Versioning](https://semver.org/lang/zh-TW/)。

## [0.12.0] - 2026-07-22

角色憑證:成員的角色由擁有者簽章、客戶端可驗,伺服器無法捏造。

### 新增

- **客戶端可驗的角色憑證**:擁有者核准成員、改角色或輪換金鑰時,都會對「這個 vault、這位成員、這個角色、這個金鑰紀元」簽出一份 Ed25519 憑證,經伺服器分發;成員取金鑰時以邀請時取得的擁有者公鑰(信任錨)驗證。**角色指派從此防竄改**——被攻陷的伺服器無法把檢視者謊稱成編輯者,補上 0.10.0 明定的信任邊界缺口。

### 安全

- **偽造即拒**:憑證簽章驗不過(偽簽、跨 vault 或跨成員挪用、內容竄改)時,成員端直接中止金鑰載入,不會拿著假角色運作。
- **舊憑證重放防護**:憑證綁定金鑰紀元,輪換金鑰即作廢整代舊憑證——伺服器重放輪換前的憑證不會被採信。同一紀元內改角色後的舊憑證重放仍屬已知限制(兩者皆為真簽),擁有者輪換金鑰即可沖洗;每筆編輯的作者驗證留待後續版本。
- **平滑升級**:升級前已核准的成員尚無憑證,維持原行為(顯示本地既知角色);擁有者下次改角色或輪換金鑰時自動補發。

### 品質

- 憑證單元測試(正簽正驗、偽簽、挪用、內容竄改、截斷)與端到端測試(核准/改角色簽發流、資料庫植入偽簽憑證即拒、輪換重簽與舊紀元重放降級、非擁有者簽發被拒、移除成員清除憑證)。全庫 429 測全綠。

## [0.11.0] - 2026-07-22

金鑰輪換:移除成員後整個 vault 換新金鑰並重新加密,離線舊金鑰從此解不開新內容。

### 新增

- **移除成員即輪換金鑰**:擁有者移除成員後,自動產生新的團隊金鑰、重新包給所有留任成員,並把 vault 內容全部以新金鑰重新加密。被移除者即使留有離線舊金鑰與舊快取,也解不開此後的任何內容——補上 0.10.0 已知限制中的密碼層前向保密。
- **手動輪換**:團隊面板新增「輪換金鑰」按鈕,自動輪換失敗(例如網路中斷)時可重試,也可作例行輪換。
- **待核准標記**:成員清單標示尚未核准的成員;輪換只把新金鑰重新包給已核准成員,不會繞過指紋核對。

### 變更

- **輪換後分享連結作廢**:分享連結內含由團隊金鑰衍生的筆記金鑰,輪換後一律失效並即時切斷已連線的分享檢視;需要時以新金鑰重新建立分享。個人 vault 的分享不受影響。

### 安全

- **紀元寫入柵欄**:每個團隊 vault 帶單調遞增的金鑰紀元(epoch),輪換 commit 後伺服器立即拒絕舊紀元的寫入——確保輪換窗口內不會有舊金鑰密文混入共享日誌造成永久污染。
- **崩潰安全**:輪換全程冪等——commit 前任何失敗都不留半套狀態(紀元不動、舊金鑰續用);commit 後若重加密中斷,擁有者重啟會自動續跑補完(每篇重加密都等伺服器確認落盤才算數)。
- **成員自癒**:線上成員收到輪換廣播即暫停推送、取回新金鑰後無縫收斂;離線錯過輪換的成員重連時發現紀元落後,同樣自動取回新金鑰並重新拉齊,期間絕不套用解不開的資料。

### 品質

- 輪換全鏈測試:伺服器層(紀元 CAS 只接受恰為當前+1、柵欄拒舊紀元寫入、輪換廣播、分享作廢與踢線)與桌面金牌(三成員經真伺服器——留任者收斂新金鑰並雙向編輯、被移除者舊金鑰解不開重加密內容、重加密冪等、離線者重連自癒、未拉齊即中止且紀元不動)。全庫 419 測全綠。

## [0.10.0] - 2026-07-21

團隊角色與授權:owner / editor / viewer,伺服器強制讀寫權,踢人立即生效。

### 新增

- **成員角色(owner / editor / viewer)**:團隊 vault 現在區分角色。**擁有者**全權並可管理成員;**編輯者**可讀寫;**檢視者**唯讀。擁有者產生邀請碼時就決定被邀者的角色,加入即生效。桌面團隊面板可顯示每位成員的角色、隨時把成員在編輯者/檢視者間切換,或移除。
- **伺服器逐訊息授權**:團隊 vault 上,只有擁有者/編輯者能寫入(含改動筆記路徑、空間等共享結構);檢視者的任何寫入一律被伺服器拒絕,但仍可讀取與即時看到協作。個人 vault 完全不受影響。
- **移除/降級立即踢線**:擁有者移除成員或把編輯者降為檢視者時,對方的**活躍連線當場切斷**——不再等到下次斷線。被移除者無法重連(邀請碼一次性、已消耗)。

### 變更

- 升級既有 0.9.0 團隊 vault 時,伺服器會自動為成員名冊補上角色欄位(既有成員預設為檢視者,擁有者維持擁有者身分);**擁有者可在團隊面板重新指派各成員的角色**。

### 安全

- **授權的信任邊界(明確界定)**:在全盲中繼的端對端加密模型下,角色的**讀寫強制是「可信伺服器」層級的控制**(防濫用、防誤操作),而非密碼學保證——檢視者本身持有團隊金鑰,一個被攻陷的伺服器理論上仍可接受其寫入。**機密性與金鑰分發**維持不信任伺服器的端對端加密(空間金鑰信封的擁有者簽章才是客戶端可驗的密碼學邊界)。把角色指派也變成客戶端可驗、以及每筆編輯的作者驗證,留待後續版本。

### 品質

- 角色全鏈測試:store 角色 CRUD 與邀請碼帶角色、協議往返;伺服器端對端(編輯者可寫 / 檢視者寫入含 vault-meta 皆拒 / 任何成員可讀、移除與降級即時踢線並擋重連、非擁有者管理指令被拒、升級自癒讓遷移後的擁有者仍能寫入)。全庫 404 測全綠。

### 已知限制(留待後續)

- **踢人尚未輪換金鑰**:移除成員只切連線與刪其金鑰信封,不重換團隊金鑰;被移除者若留有離線舊快取,仍能解先前已同步的舊內容(對「新內容」則因無法重連而取不到)。密碼層的前向保密(換金鑰 + 重加密)為下一階段。

## [0.9.0] - 2026-07-21

團隊協作正式落地:多人各以自己的身分,共享同一個加密 vault。

### 新增

- **團隊 vault(邀請 + 空間金鑰盲中繼)**:一個團隊 vault 有一把隨機的**團隊金鑰**,用每位成員的公鑰包成信封,經**全盲伺服器**轉交;成員以自己的身分金鑰解出後即可協作。伺服器看不到金鑰明文,也讀不到內容。加入流程:擁有者產生**一次性邀請碼**交給對方(out-of-band),對方憑碼加入後,擁有者**核對其公鑰指紋**再核准(把團隊金鑰包給他)。核准前成員處於「等待授權」狀態,不會誤讀任何東西。擁有者可**移除成員**。桌面新增團隊面板:建立 / 加入團隊 vault、產生邀請碼、成員清單與指紋核對、核准 / 移除。
- **身分種子上 OS keychain**:成員身分的種子改由系統金鑰鏈(macOS Keychain / Windows DPAPI)加密後才落盤;keychain 不可用時優雅退回、並相容既有明文檔。因為這把種子的洩漏面在團隊時代跨越多個 vault,值得比 vault 密語更強的保護。

### 變更

- **團隊 vault 強制身分認證**:設有擁有者的 vault 一律走 challenge-response 身分認證,拒絕只帶 token 的舊式連線,並擋下非成員的寫入——順帶堵掉「任何持 token 者上傳一份垃圾快照即可截斷他人真實編輯」的可用性攻擊。個人 vault 的密語同步完全照舊,零改變。

### 安全

- **信封由擁有者簽章 + 情境綁定**:空間金鑰信封帶擁有者的 Ed25519 簽章,成員以 out-of-band 已知的擁有者指紋驗簽——擋下「惡意伺服器自造信封 + 自造中繼資料,餵新成員一個偽造的 vault」。信封的加密情境綁定 vault / 金鑰 / 版本 / 收件人,防跨情境挪用重放;解封拒絕退化的金鑰交換結果。

### 品質

- 團隊全鏈測試:金鑰信封 wrap/unwrap(含偽造、竄改、跨情境、退化點防線）、協議往返、伺服器授權(邀請碼單次 / 綁 vault / 過期、擁有者專屬操作、信封只回本人、team vault 拒舊式連線)、bootstrap 死結防護與 pending→ready、擁有者管理連線、邀請 bundle 編解碼、身分種子 keychain 往返;**金牌**:兩個不同身分的成員經真伺服器雙向收斂、以團隊金鑰加解密互通。全庫 393 測全綠。

### 已知限制(留待後續)

- **踢人後金鑰未輪換**:移除成員只刪其名冊列與信封,不重換團隊金鑰;被移除者若留有舊金鑰快取,仍能離線解舊內容。真正的踢人(輪換 + 重包給留任者）與 owner/editor/viewer 角色強制留待下一階段。
- **per-space 成員子集**:本版團隊是「整個 vault 共享」,尚無「某空間只給部分人」。資料模型已為此預留。

## [0.8.0] - 2026-07-21

本地優先補完 + 團隊協作的身分地基:空間與留言不再需要同步,並為團隊功能鋪好成員身分。

### 新增

- **成員身分(團隊協作的地基)**:每台裝置持有一把長期的**非對稱成員身分**——單一隨機種子衍生出簽章(Ed25519)與金鑰包裝(X25519)兩把子金鑰。連線時對伺服器做 **challenge-response**(簽名綁定伺服器亂數 + vault + 成員,防重放與跨 vault 挪用),證明「你是誰」而不只是「你知道 token」。伺服器把成員公鑰記進 per-vault 名冊(公鑰**首見即釘選**,擋冒名)。身分獨立於任何 vault 密語,存本機獨立檔、可匯出匯入到其他裝置。**這是團隊空間的樞紐**:未來邀請成員 = 用對方的公鑰把空間金鑰包起來經全盲伺服器轉交,難題從「不可能」降到「可做」。個人使用者無感——現有 vault 的密語同步完全照舊。

### 變更

- **空間與留言不再需要啟用同步**:先前「空間分組」與「留言討論」都掛在同步層管理的中繼資料上,未開同步時整組退化(側欄變平面清單、留言面板停用)。現在把中繼資料的所有權從同步層抽離,交由 vault 本身持有——**純本地、沒開同步的 vault 一樣能建空間、移動/複製筆記、留言討論**,同步啟用時才額外把它們推上雲。這修正了本地優先產品不該有的破口。

### 品質

- 身分全鏈測試:金鑰生成/簽章/序列化 6 例、協議往返、伺服器 challenge-response 端對端(含搶註、重放、跨裝置匯入)、桌面身分檔持久化;純本地空間 5 例 + 純本地留言 2 例守住「無同步也可用」不變量。
- 安全審查(密碼學 + 認證):修掉一個搶註漏洞(伺服器綁定成員 id 為其公鑰雜湊,擋任意 id 配自帶金鑰霸佔他人身分),並硬化身分檔落盤(原子寫、壞檔不靜默換身分)。桌面測試增至 97,全庫 347 測全綠。

### 已知限制

- 團隊空間本身(邀請、角色、用公鑰包裝空間金鑰)尚未開放——本版只鋪身分地基,伺服器記錄成員但**尚不以成員身分做授權**,存取仍靠既有 token + 密語。成員名冊在本版是諮詢性紀錄。

## [0.7.0] - 2026-07-20

空間與權限地基:把筆記分組到帶金鑰的「空間」,為團隊協作鋪路。

### 新增

- **空間(vault → 空間 → 筆記三層地基)**:側欄可把筆記分組到「空間」。每個空間是一級「帶金鑰單元」——空間金鑰往下衍生每篇筆記的金鑰;**預設空間沿用主金鑰,既有 vault 零遷移、金鑰一位元組不變**。可建立、改名空間(預設空間也可改名),分組可折疊。這是團隊/組織協作的架構地基:空間金鑰的取得已抽象,未來團隊空間只換「金鑰來源」(改為隨機生成 + 成員公鑰包裝),加解密與同步下游不動。
- **跨空間移動與複製**:移動筆記到別的空間會跨金鑰邊界——確認後以新空間金鑰重新加密,並重推快照截斷舊金鑰增量,讓其他裝置以新金鑰解出(跨裝置收斂)。複製則在目標空間生一篇**全新筆記**(新 docId、自出生即以目標空間金鑰加密),原筆記原封保留。
- **空間變更稽核紀錄**:建立、改名、移動、複製都留 append-only 事件,側欄可查(單人「好追查」,團隊層將成合規稽核命脈)。
- **Homebrew 一行安裝**:`brew install --cask cluion/tap/stele`;發版 CI 自動更新 tap 的 Cask 版本與雜湊。未簽章,首次開啟右鍵 Stele.app → 開啟。

### 品質

- 空間全鏈測試:金鑰衍生 7 例(含「預設空間與舊金鑰位元組相同」零遷移守門、新空間兩層互不可解)、空間模型 9 例(登記/歸屬/稽核/CRDT 合併)、真伺服器端對端 4 例(零遷移同步、跨裝置移動收斂、後加入裝置 bootstrap、複製雙篇收斂)。以真 Electron 端對端金牌驗證「建立空間 → 側欄分組 → 移動筆記 → 稽核記錄」。

### 已知限制

- 空間目前需啟用加密同步(空間資料掛在 vault-meta,由同步層管理);未啟用同步的 vault,側欄退回平面清單。與留言同一約束。

## [0.6.0] - 2026-07-18

留言與討論:錨定到文字範圍的協作討論串,獨立資料型別、不污染筆記。

### 新增

- **留言與討論**:在編輯器選取文字即可留言,他人可回覆成討論串、可標記已解決或刪除。留言錨定到文字範圍(Yjs relative position),你我插字時錨點自動位移,原文被刪則標「原文已刪」。源碼模式字元級高亮、所見即所得模式整段高亮;留言面板列出全篇討論、點引用即捲到原文。
- **留言是獨立的伴生資料**:每篇筆記的留言存在與筆記本體不同的 CRDT doc,**不進 .md 鏡像、不污染筆記歷史**;走同一條端對端加密同步,留言 doc 自得一把獨立子金鑰,伺服器仍全盲。留言 doc 的 id 由筆記 id 決定性衍生,兩裝置各自算出同一個,免對照表也不會併發分裂。

### 品質

- 留言全鏈測試:討論串模型與範圍錨定 6 例(含 CRDT 合併、錨點位移/塌縮)、伴生 doc 真伺服器端對端 2 例(跨裝置留言/回覆同步、對方解得出錨定、留言不進 .md);桌面 smoke 增至 19。以真 Electron 端對端驗證「選取文字→留言→同步到第二裝置→對方解出錨定」完整往返。

### 已知限制

- 留言需啟用加密同步(伴生 doc 由同步層管理);未啟用同步的 vault,留言面板會提示而非靜默失效。

## [0.5.0] - 2026-07-18

可編輯分享閉環:貼上分享連結,在桌面開臨時協作視窗即時共編。

### 新增

- **消費可編輯分享連結**:桌面貼上別人給的分享連結(側欄 ⇲ 鈕、歡迎畫面入口或 Cmd/Ctrl+Shift+O),即以「臨時協作視窗」開啟。可寫連結能即時編輯,內容即時同步回來源方 vault;唯讀連結以唯讀開啟。**消費者不需要自己的 vault**,關掉視窗即結束,本地不留檔。分享功能自此雙軌俱全:唯讀走瀏覽器檢視器,可編輯交桌面協作。
- **端對端加密不打折**:金鑰只從連結 fragment 取得且留在桌面主行程(不進有 IPC 面的 renderer 主世界);伺服器仍全盲只中繼密文;唯讀權限由伺服器回報並在編輯器層鎖定輸入。

### 品質

- 新增消費端測試:連結解析 4 例、SharedSession 真伺服器端到端 3 例(可寫回寫來源磁碟、唯讀不外洩、撤銷即時失效);桌面 smoke 增至 18 項。以真 Electron 端到端驗證「貼連結→協作視窗打字→回寫來源磁碟」完整往返。
- 修穩 CI:補 `packageManager` 讓 GitHub Actions 的 pnpm setup 認得版本、smoke 於 CI 固定 zh-TW locale,發版與 CI 兩條 workflow 皆綠。

## [0.4.0] - 2026-07-18

桌面安裝檔與自動發版:一鍵下載的 dmg / AppImage / deb,推 tag 即自動打包並發佈。

### 新增

- **桌面安裝檔打包**:`apps/desktop` 接上 electron-builder,產出 macOS dmg(x64 + arm64 雙架構)、Linux AppImage 與 deb,附石碑應用圖示。產物走 esbuild 全 bundle,asar 只收 `dist/` 不含 node_modules,dmg 檔名帶架構後綴避免雙架構互相覆蓋。
- **CI 自動發版**(新 `.github/workflows/release.yml`):push `v*` tag 觸發,`build` job 在 macOS 與 Linux runner 並行打包三平台安裝檔,`release` job 收齊產物後從 `CHANGELOG.md` 抽出對應版本節當 release notes,`gh release create` 一次帶齊安裝檔直接發佈;發佈前斷言 tag 版本與 `package.json` 一致,擋下忘記升版號的誤發。`workflow_dispatch` 另備只建置不發佈的 dry-run。

### 品質

- 發版流程全自動化,原本手動的跨平台打包與建 Release 交由 CI 完成;Linux 產物改由 Linux runner 實際打包,不再是 mac 上跨建。發版流程文件(`plan/RELEASE.md`)同步更新。

## [0.3.0] - 2026-07-17

分享連結:把單一筆記唯讀分享到瀏覽器,伺服器仍全盲。

### 新增

- **唯讀分享連結**:右鍵任一筆記即可建立分享連結,對方用瀏覽器就能開,免安裝、免帳號。桌面對話框含複製連結與撤銷清單。
- **網頁檢視器**(新 `apps/viewer`):與桌面同一套 editor-core schema 唯讀渲染,連結、表格、callout 呈現一致。伺服器同埠掛靜態頁,`shareId` 只在前端解析。
- **端對端加密延伸到分享**:每則分享匯出「單一 doc 金鑰」(HKDF 單向,vault 主金鑰不外洩),金鑰只走 URL fragment(`#` 之後)不進伺服器;伺服器新增 shares 表管理作用域與撤銷,唯讀連線鎖定單一 doc 且拒所有寫入型訊息。

### 修正

- **撤銷即時生效**:撤銷分享會當場切斷既有連線,撤銷後的內容一個字都拿不到(原本只擋新連線,已連上的可續讀);且撤銷嚴格綁 vault,猜中他人 `shareId` 也踢不掉對方連線。
- **連結 href 淨化**:編輯器與檢視器只放行 `http`/`https`/`mailto` 與相對連結,`javascript:`/`data:` 等一律剝除(貼上 HTML 會繞過 markdown-it 的內建過濾,渲染器主世界有 IPC 面,不能只靠上游解析器設定)。
- **檢視器安全標頭**:分享頁補上 CSP(`default-src 'none'`、腳本只信同源、`frame-ancestors 'none'`)、`nosniff`、`no-referrer`;`frame-ancestors` 下在 HTTP header,`<meta>` 版另備為自架保底。

### 品質

- 分享全鏈測試:金鑰匯出/單 doc 解密、協議往返、伺服器作用域與權限、撤銷即時與跨 vault 隔離、桌面「建立→列出→撤銷」真實往返(驗證 fragment 金鑰可還原且不落伺服器)、檢視器連結解析。
- 桌面 smoke 增至 17 項,新增分享對話框開啟/建立/關閉全鏈。

## [0.2.0] - 2026-07-17

即時多人協作:加密的在場指示與遠端游標。

### 新增

- **即時協作 awareness**:多人開同一 vault 時,即時顯示誰在看哪篇筆記(彩色頭像,顏色由裝置穩定衍生)。
- **遠端游標**:兩種編輯器模式都渲染協作者游標——源碼模式為字元級精確 caret 與選取範圍(Y.Text relative position,並發編輯下不漂移);所見即所得模式為塊級段落標記與名字標籤。
- **端對端加密**:awareness 與游標資料以同一把 doc 金鑰加密,伺服器為 blind relay,看不到誰在哪、也不落盤(ephemeral)。

### 修正

- 本地游標回報加上節流(~90ms),打字期間不再每按鍵一次加密廣播。
- 協作熱路徑改用編輯器增量維護的區塊映射,不再每次按鍵/在場更新整份重新解析 Markdown。
- 同步引擎:awareness 對非法 docId 拒絕(防資源耗盡)、切 vault 重置在場、stop 後不復活實例。

## [0.1.0] - 2026-07-17

首個公開版本:本地優先單機 MVP + 端對端加密同步 + 自架伺服器。

### 新增

- **編輯器核心**:ProseMirror 真 WYSIWYG + CodeMirror 6 源碼模式,塊級重序列化保位元組穩定;不支援語法走 opaque 原文節點。
- **Wikilink 生態**:`[[ ]]` 自動完成、點擊導航、就地建檔、改名連動全庫改寫。
- **雙鏈**:反向連結面板、自寫力導向關聯圖。
- **每日筆記**:`{{date}}` 模板、`.stele/config.json` 可自訂。
- **搜尋與切換**:CJK bigram 全文搜尋、快速切換器。
- **設計系統**:日石英 / 夜燭石 雙主題(原生設計,非反轉)。
- **CRDT 狀態持久化**:`.stele/` 保存二進位狀態,開檔吸收外部差異,歷史跨啟動延續。
- **端對端加密同步**:passphrase → scrypt → 主金鑰,per-doc HKDF 子金鑰 + AES-256-GCM;伺服器只見密文。
- **自架同步伺服器**:Node + ws + better-sqlite3 的 blind relay,單 Docker 一行起服;雙路徑同步(快照 + 增量)、離線佇列、路徑 LWW。
- **i18n**:zh-TW / en,CI 缺譯檢查。

### 品質

- 209 個單元/整合測試,含 8 輪固定 seed 多裝置混沌測試(隨機編輯/刪除/斷線/中殺,零資料遺失為底線)。
- 16 項 smoke 測試涵蓋桌面全鏈。
- CI:lint、typecheck、test、授權政策檢查、smoke。
